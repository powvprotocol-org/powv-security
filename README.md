# Security Policy (PoWV Security)

---

##  Supported Versions

The PoWV Protocol is currently in pre-release development.  
At this stage, there are no public production releases or versioned software distributions. Security reviews are conducted internally as part of ongoing architecture and protocol development.

---

##  Reporting a Vulnerability

If you believe you have identified a security vulnerability related to the PoWV Protocol, please report it responsibly. 

When submitting a report, please include:
* **Description:** A detailed description of the security issue.
* **Potential Impact:** The theoretical or practical impact on the protocol.
* **Reproduction Steps:** Steps to reproduce the vulnerability (if applicable).
* **Proof of Concept:** Code snippets or logs demonstrating the issue (if available).

### Contact Information
Security reports must be sent directly to:  
 **powv.protocol@proton.me** / **gabriel@powvprotocol.org**

*(Temporary contact channels may be utilized until dedicated security infrastructure is fully established).*

---

##  Disclosure Policy

Please do not disclose vulnerabilities publicly prior to coordinated review. The PoWV Protocol adheres to responsible disclosure principles. 

Valid reports will be reviewed and evaluated based on:
* Technical severity
* Exploitability
* Protocol-level impact
* Infrastructure risk

Our goal is to acknowledge and assess valid reports within a reasonable timeframe.

---

##  Scope

Security considerations and vulnerability assessments include, but are not limited to:
* Core protocol architecture
* Cryptographic mechanisms & Hardware root-of-trust
* Consensus logic verification
* Smart contract vulnerabilities
* Oracle attack vectors & telemetry tampering
* Infrastructure trust boundaries
